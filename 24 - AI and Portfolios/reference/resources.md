# Curated Resources for Data Science Portfolio Building

Reading list organized by topic. These are resources that will inform your portfolio strategy and help you stay current on what hiring managers care about in 2026.

## AI-Powered Data Analysis

Tools and approaches for incorporating AI into your analytical workflow without losing clarity or control.

- **Navigating the Jagged Technological Frontier (Dell'Acqua et al., 2023)** - https://papers.ssrn.com/sol3/papers.cfm?abstract_id=4573321
  The BCG/Harvard study that found structured AI use improved performance by 40%, while unstructured use decreased it by 19%. This is the foundation for the two-pass workflow we use in this session.

- **Student Guide to AI-Assisted Learning (Mark Keith, BYU IS)** - Provided separately
  Framework for engaging with AI across eight modes in three tiers (Passivity, Partnership, Agency). Source of the Prep Mode and Challenge Mode concepts used in our spec-writing activity.

- **Exploring Gen AI: Tools (Martin Fowler)** - https://martinfowler.com/articles/exploring-gen-ai/sdd-3-tools.html
  How to use AI assistants as analytical partners, with emphasis on maintaining rigor and transparency.

- **Introduction to Prompt Engineering for Data Professionals (Dataquest)** - https://www.dataquest.io/blog/introduction-to-prompt-engineering-for-data-professionals/
  Practical guide to structuring requests to LLMs for data analysis, exploration, and documentation tasks.

- **Data Analysis Pipeline with Claude Code (Medium/AiGuys)** - https://medium.com/aiguys/data-analysis-pipeline-with-claude-code-2-486-medium-articles-to-interactive-insights-45d39417334b
  End-to-end example of building a reproducible analysis pipeline with AI assistance.

## Tools and Platforms Reference

Quick reference for popular tools used in portfolio projects. Best used for, and when to reach for them.

| Tool | Best For | Notes |
|------|----------|-------|
| **Claude** | Brainstorming, code review, documentation, exploratory analysis | Free tier available via claude.ai. Excellent for ideation and thinking through problems. API available for production use. |
| **ChatGPT** | General-purpose AI assistance, rapid prototyping | Widely familiar. Good for quick answers. Code generation is solid but Claude tends to be stronger for data work. |
| **GitHub Copilot** | Code generation and completion | Integrates directly into your editor. Great for speeding up routine coding tasks. Subscription required. |
| **GitHub Pages** | Hosting portfolio website | Free static hosting. Works with Jekyll or custom HTML. Easy to set up. No backend. |
| **Quarto / R Markdown** | Creating portfolio documentation and websites | Quarto is the modern standard. Renders to HTML, PDF, or Word. Great for mixing code, output, and narrative. |
| **Streamlit** | Interactive dashboards and demos | Fast prototyping of interactive data apps. Deploy for free on Streamlit Cloud. Great for Tier 3 projects. |

### When to Choose What

- **For a polished website showcasing multiple projects:** Use GitHub Pages with Quarto or custom HTML. Keep it simple and fast.
- **For an interactive demo of your work:** Use Streamlit. Easier than building a web app from scratch, and deployment is simple.
- **For project documentation and analysis writeups:** Use Quarto. Mix code, output, and prose in one file. Renders beautifully.
- **For coding help during development:** Use GitHub Copilot (integrated into your editor) and Claude or ChatGPT in separate windows.
- **For brainstorming and thinking through approach:** Use Claude. Work through problems conversationally before you code.

## How to Use This List

Start with the Portfolio Strategy section. Those three articles will give you a clear picture of what hiring managers want to see and what most people get wrong.

Then read one article from Career and Future of Data Work to understand what the job market looks like in 2026. This will inform which portfolio projects you choose.

As you build, bookmark the AI-Powered Data Analysis links. These will help you work faster and stay methodical as you use AI tools in your projects.

The tools table is just a reference. You do not need all of these. Pick one way to host your portfolio (GitHub Pages or Streamlit or both) and one tool for building it (Quarto or hand-coded HTML). Keep your setup simple. The work matters more than the presentation.
